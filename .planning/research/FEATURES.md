# Project Research: Features

## Table Stakes

### Course Overview

- Clear course promise: from block commands to real hardware.
- Duration and structure: parts, lesson count, 60-90 minute session range.
- Beginner-friendly audience statement.
- Final outcomes: smart-home or robot model.

### Lesson Cards

Each lesson should expose:

- lesson title
- learning goal
- theory concepts
- practice activity
- core Scratch/mBlock blocks or hardware components
- expected output

### Learning Progression

Scratch progression should follow common beginner CS sequencing:

1. interface, events, first action
2. sequence and timing
3. forever loops
4. repeat loops
5. conditionals and sensing

Hardware/mBlock progression should move from output to input to sensing to motion:

1. Arduino/mBlock setup and LED digital output
2. traffic light with multiple LEDs and loops
3. button digital input and if/else
4. ultrasonic sensor and variables
5. servo and PWM/angle control
6. custom blocks/functions
7. capstone combining sensor + servo

### Method Pillars

- AI as a helper for explanation, debugging prompts, and reflection.
- draw.io flowcharts before code for algorithm thinking.
- Project-based learning with visible artifacts.
- Hardware safety/wiring basics.

## Differentiators

- Connects Scratch concepts directly to physical electronics equivalents.
- Uses flowcharts as a bridge between block logic and hardware behavior.
- Shows sponsor/education impact through a credible curriculum, not just a course list.
- Offers two capstone choices: smart trash bin or reverse parking warning system.

## Anti-Features

- Do not turn this into a dense syllabus PDF embedded on a page.
- Do not present every bullet at equal weight; beginners need a guided path.
- Do not make AI the subject of the course; AI supports problem-solving.
- Do not introduce too many hardware parts before students understand digital input/output.

## Complexity Notes

- **Low:** Static overview, lesson cards, timeline/stepper.
- **Medium:** Interactive filters/tabs by Scratch vs Hardware.
- **Medium:** Visual flowchart examples and downloadable resources.
- **High:** LMS progress, quizzes, accounts, hardware simulator.
