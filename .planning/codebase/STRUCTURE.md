# Codebase Structure

## Root Layout

- `package.json`: scripts and dependencies.
- `package-lock.json`: npm lockfile.
- `next.config.js`: minimal Next config.
- `tailwind.config.ts`: Tailwind theme extension.
- `postcss.config.mjs`: PostCSS/Tailwind entry.
- `eslint.config.mjs`: ESLint flat config.
- `tsconfig.json`: TypeScript compiler and `@/*` path alias.
- `components.json`: shadcn component config.
- `README.md`: long bilingual project spec and implementation guide.
- `PRODUCT.md`: strategic product/brand context generated by Impeccable.
- `DESIGN.md`: visual design system generated by Impeccable.

## Source Directory

### `src/app/`

Next App Router routes and major page sections.

- `src/app/layout.tsx`: global shell.
- `src/app/page.tsx`: primary homepage route.
- `src/app/sponsor/page.tsx`: sponsor route.
- `src/app/globals.css`: theme tokens, global Tailwind imports, custom animations.
- `src/app/1. competition/`: competition/FIRST/FTC/hero sections.
- `src/app/2. team/`: team, mission, achievements, challenge sections.
- `src/app/3. financial-need/`: budget and fundraising sections.
- `src/app/4. sponsorship/`: sponsor process and sponsor value sections.
- `src/app/shared/`: page-level shared sections such as `FinalCTA`.

### `src/components/`

Reusable components.

- `src/components/layout/`: `Navbar`, `Footer`, and `DynamicMetadata`.
- `src/components/shared/`: marketing-specific components such as `CTAButton`, `GlowCard`, `ImageGallery`, `MediaPlaceholder`, `SectionHeader`, `Timeline`, `PackageCard`, `BudgetTable`, and `ContactForm`.
- `src/components/ui/`: shadcn/Radix-style primitives and custom primitives such as `aurora-background`, `glass-button`, `member-card`, and `bento-monochrome`.
- `src/components/magicui/`: animated/special-effect components such as `MagicCard` and `BorderBeam`.

### `src/data/`

Static content modules.

- `src/data/hero.ts`
- `src/data/aboutFIRST.ts`
- `src/data/aboutFTC.ts`
- `src/data/about.ts`
- `src/data/team.ts`
- `src/data/achievements.ts`
- `src/data/mission.ts`
- `src/data/budget.ts`

### `src/contexts/`

- `src/contexts/LanguageContext.tsx`: locale state, message loading, translation lookup.

### `src/hooks/`

- `src/hooks/useTranslatedData.ts`: picks `_vi` / `_en` fields from data objects.
- `src/hooks/useDynamicMetadata.ts`: dynamic metadata helper.

### `src/lib/`

- `src/lib/utils.ts`: utility helpers, especially `cn()` for class merging.

## Messages

- `messages/vi.json`: Vietnamese UI strings.
- `messages/en.json`: English UI strings.

## Public Assets

- `public/Images/**`: local raster images grouped by section/domain.
- `public/Images/ASSETS.md`: asset inventory notes.
- `public/Logo/**`: Robotics Soc Son SVG logo assets.

## Documents

- `Documents/Sponsorship.md`, `Documents/Sponsorship2.md`, `Documents/Sponsorship_Complete.md`: sponsorship source materials.
- `Documents/Thành tích.md`: achievement content source.
- `Documents/Dự trù kinh phí.md`: budget source.

## Scripts

- `scripts/check-assets.mjs`: verifies that `/Images/...` references in source files exist under `public/`.

## Naming Conventions

- Section folders under `src/app/` use numbered prefixes to define narrative order.
- Component files use PascalCase for React components (`CTAButton.tsx`, `GlowCard.tsx`) and lowercase for shadcn-style primitives (`button.tsx`, `card.tsx`).
- Data fields commonly use bilingual suffixes: `title_vi`, `title_en`, `description_vi`, `description_en`.
- Public image paths contain spaces and Vietnamese characters in some filenames; `MediaPlaceholder` normalizes/encodes filenames when needed.

## Important Entry Points For Changes

- Add/reorder homepage sections: edit `src/app/page.tsx`.
- Update global brand tokens: edit `src/app/globals.css` and keep `DESIGN.md` aligned.
- Update navigation labels: edit `messages/vi.json`, `messages/en.json`, and `src/components/layout/Navbar.tsx` if routes/anchors change.
- Update section content: edit the matching `src/data/*.ts` file.
- Add image assets: place under `public/Images/**`, reference via `/Images/...`, then run `npm run check:assets`.
